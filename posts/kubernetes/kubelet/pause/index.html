kubernetes 源码分析之 Pause 容器

Created: Jan 27, 2025 · 1 min read

本篇文章基于 Kubernetes v1.25.1

Pod 是 kubernetes 里最小调度单元，一个 Pod 里面可以包含由多个容器。 这些容器之间通过 linux namespace 来共享资源以及和宿主节点环境隔离。Pod 里面最基础的容器是 pause 容器，它是 Pod 启动的时候，首先创建的容器，其他的业务容器都依赖该 pause 容器，毕竟它提供了可共享的 linux namspace。

Pause 容器的代码不到 100 行：

static void sigdown(int signo) {
  psignal(signo, "Shutting down, got signal");
  // 直接正常退出进程
  exit(0);
}

static void sigreap(int signo) {
  // waitpid 监听子进程退出, WNOHANG 是非阻塞标记, 当没有找到子进程退出时, 不会阻塞
  while (waitpid(-1, NULL, WNOHANG) > 0)
    ;
}

int main(int argc, char **argv) {
  int i;
  for (i = 1; i < argc; ++i) {
    if (!strcasecmp(argv[i], "-v")) {
      printf("pause.c %s\n", VERSION_STRING(VERSION));
      return 0;
    }
  }

  if (getpid() != 1)
    /* Not an error because pause sees use outside of infra containers. */
    fprintf(stderr, "Warning: pause should be the first process\n");

  // 注册信号和回调方法
  // SIGINT 和 SIGTERM 都是正常退出，参考上面的回调函数sigdown
  if (sigaction(SIGINT, &(struct sigaction){.sa_handler = sigdown}, NULL) < 0)
    return 1;
  if (sigaction(SIGTERM, &(struct sigaction){.sa_handler = sigdown}, NULL) < 0)
    return 2;
  // SIGCHLD ,回收子进程
  if (sigaction(SIGCHLD, &(struct sigaction){.sa_handler = sigreap,
                                             .sa_flags = SA_NOCLDSTOP},
                NULL) < 0)
    return 3;
  for (;;)
    pause(); // 进程进入休眠，监听上诉注册信号的发生
  fprintf(stderr, "Error: infinite loop terminated\n");
  return 42;
}

从上面的代码可以看出 pause 容器做如下事情：

  • 注册进程退出信号和子进程相关信号：即收到 SIGINT/SIGTERM 后，pause 进程直接退出；收到 SIGCHLD 信号，则调用 waitpid 进行子进程的回收。
  • 循环调用 pause() 函数，使进程陷入休眠状态，当监听到注册信号后触发回调函数处理逻辑。
