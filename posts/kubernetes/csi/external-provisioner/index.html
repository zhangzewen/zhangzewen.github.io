kubernetes csi 之 external-provisioner

Created: Aug 2, 2025 · 2 min read

前言

external-provisioner 是一个控制器，通过监控 PVC/PV 资源的变化来创建和删除 PV。 其主要逻辑已由 sig-storage-lib-external-provisioner 实现。唯一需要做的是实现接口 Provisioner 来提供 Volume 的创建和删除，因为 PVC 和 PV 的绑定是 PV Controller 的工作。

type Provisioner interface {
	Provision(context.Context, ProvisionOptions) (*v1.PersistentVolume, ProvisioningState, error)
	Delete(context.Context, *v1.PersistentVolume) error
}

Volume 的创建和删除：csiProvisioner

创建 Volume

csiProvisioner.Provision 的代码很多，但是流程清晰：

  • 通过 CSI 接口 提供的 Controller Service 和 CSI plugin 通信（调用CreateVolume）来创建 Volume。
  • CSI plugin 创建 volume 成功返回后，构建 PV 资源对象。
  • 把PV 资源对象 交给 sig-storage-lib-external-provisioner 处理。

func (p *csiProvisioner) Provision(ctx context.Context, options controller.ProvisionOptions) (*v1.PersistentVolume, controller.ProvisioningState, error) {
	claim := options.PVC
    //通过检查PVC的.metadata.annotations的键"volume.kubernetes.io/storage-provisioner"/"volume.beta.kubernetes.io/storage-provisioner"
    // 来获取provisioner
	provisioner, ok := claim.Annotations[annStorageProvisioner]
	if !ok {
		provisioner = claim.Annotations[annBetaStorageProvisioner]
	}
    // 需要注意的是p.driverName,在main函数的时候通过CSI gRPC调用来获取的drivername
    // 还需要判断.metadata.annotations["pv.kubernetes.io/migrated-to"]的值是不是driverName
	if provisioner != p.driverName && claim.Annotations[annMigratedTo] != p.driverName {
		// The storage provisioner annotation may not equal driver name but the
		// PVC could have annotation "migrated-to" which is the new way to
		// signal a PVC is migrated (k8s v1.17+)
		return nil, controller.ProvisioningFinished, &controller.IgnoredError{
			Reason: fmt.Sprintf("PVC annotated with external-provisioner name %s does not match provisioner driver name %s. This could mean the PVC is not migrated",
				provisioner,
				p.driverName),
		}
	}
    // ...
    // gRPC通信，通过 CSI plugin 创建 Volume
	rep, err := p.csiClient.CreateVolume(createCtx, req)
    // ...
    // 创建成功后，构建 pv 资源对象
	pv := &v1.PersistentVolume{
		ObjectMeta: metav1.ObjectMeta{
			Name: pvName,
		},
		Spec: v1.PersistentVolumeSpec{
			AccessModes:  options.PVC.Spec.AccessModes,
			MountOptions: options.StorageClass.MountOptions,
			Capacity: v1.ResourceList{
				v1.ResourceName(v1.ResourceStorage): bytesToQuantity(respCap),
			},
			// TODO wait for CSI VolumeSource API
			PersistentVolumeSource: v1.PersistentVolumeSource{
				CSI: result.csiPVSource,
			},
		},
	}
    //  返回该资源对象
	return pv, controller.ProvisioningFinished, nil
}

删除 Volume

csiProvisioner.Delete 通过 CSI 接口 提供的 Controller Service 和 CSI plugin 通信（调用DeleteVolume）来删除 Volume：
