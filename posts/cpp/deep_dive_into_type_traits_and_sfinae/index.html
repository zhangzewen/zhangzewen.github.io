C++ Type Traits 和 SFINAE

Created: Mar 15, 2025 · 3 min read

Type Traits

c++ type traits 特性一句话总结：“在编译期利用模板的(偏)特化功能提取目标的相关信息，用于确定策略或者实现细节”。

这在模板编程中很有用， 因为模板参数T的存在，实现模板的人很难预测使用模板的人会输入什么类型的参数，一般的 c++ type traits 的编程套路就是：

  • 一个 primary template 为通用实现
  • 多个特例化对应不同预期输入的实现逻辑
  • 然后使用一个方法封装该实现

以下代码就是一个简单的示例，判断一个是否是浮点数据类型：

#include <iostream>

template <typename T>
struct is_floating_point {
  static const bool value = false;
};

template <>
struct is_floating_point<float> {
  static const bool value = true;
};

template <>
struct is_floating_point<double> {
  static const bool value = true;
};

template <>
struct is_floating_point<long double> {
  static const bool value = true;
};

template <typename T>
void process_real_number(T const& value) {
  static_assert(is_floating_point<T>::value);
  std::cout << "processing a real number: " << value << std::endl;
}

int main() {
  static_assert(is_floating_point<float>::value);
  static_assert(is_floating_point<double>::value);
  static_assert(is_floating_point<long double>::value);
  static_assert(!is_floating_point<int>::value);
  static_assert(!is_floating_point<bool>::value);
  static_assert(!is_floating_point<long>::value);
  process_real_number(42.0);
  // process_real_number(42);  ===> error: static assertion failed
}

SFINAE

SFINAE 全称是 Substitution Failure is not an Error， 即替换失败不是错误。简单来说就是让一段代码逻辑对特定的数据类型失效，即不会在编译的时候因为数据类型的不匹配而编译错误。这个特性在模板编程中常见。 SFINAE 通常发生在模板参数推导阶段 和 重载裁决阶段。

先来看一个简单的代码示例:

#include <iostream>

struct MyType {
  using type = int;
};

class MyClass {
public:
  template <typename T>
  static void f(T x, typename T::type y) {  // #1
    std::cout << "fire" << std::endl;
  }
  template <typename T>
  static void f(T x, typename T::other_type y) { // #2
    // ...
    std::cout << "won't be fired" << std::endl;
  }
};


int main() {
 MyClass::f(MyType(), 10);
}
//output:
// fire

当调用MyClass::f(MyType(), 10) 时，编译器需要知道调用的是上述两个重载中的哪一个。实际编译流程肯定很复杂，这里简化一下流程， 首先把 MyType实例 带入到函数模板#1，很合适。然后在带入到#2 的时候发现 MyType 结构体并没有 other_type 这个类型，请注意这里说的 带入/替换 其实就是模板参数推导的过程。由于 MyType 带入到 #2 失败，从上面的代码示例可以看出在编译的过程中并没有出现编译终止报错，且运行成功。这就是 SFINAE 特性的意义所在了：即编译器在一堆集合中找最合适的结果，不合适的就跳过（SFINAE 发挥作用），当然如果最终合适的结果有多个，或者没有一个合适的，编译肯定是会报错的。

使用 enable_if 实现 SFINAE

SFINAE 通常是搭配 c++ type traits 特性一块使用的，最常用的是通过 std::enable_if 来从重载集合中剔除不合适的，其可能的实现如下：

template <bool B, typename T = void>
struct enable_if {};

template<typename T>
struct enable_if<true, T> {using type = T;};

可以看到 primary template 有一个 非类型模板参数 B 和一个 具有默认类型void的模板参数 T， 这是一个空类。有实现了一个模板参数B为true的偏特化版本，定义了一个成员类型type。所以 std::enable_if 是结合布尔表达式一块使用的， 当布尔表达式为true的时候，它就有一个成员类型type， 当布尔表达式为 false的时候，就是一个空类。

std::enable_if 可以用在：

  • 函数的返回值
  • 具有默认值的模板参数
  • 具有默认值的函数参数

以上面的判断浮点数的代码为例：

#include <iostream>
#include <type_traits>

template <typename T,
          typename std::enable_if<std::is_floating_point_v<T>>::type* = nullptr>
void f(T const& x) {  // #1
  std::cout << "float type" << std::endl;
}

template <typename T, typename std::enable_if<
                          !std::is_floating_point_v<T>>::type* = nullptr>
void f(T const& x) {
  std::cout << "not float type" << std::endl;
}

template <typename T>
void h(T const& x,
       typename std::enable_if<std::is_floating_point_v<T>>::type* = nullptr) {
  std::cout << "float type" << std::endl;
}

template <typename T>
void h(T const& x,
       typename std::enable_if<!std::is_floating_point_v<T>>::type* = nullptr) {
  std::cout << "not float type" << std::endl;
}

template <typename T>
typename std::enable_if<std::is_floating_point_v<T>>::type k(T const& x) {
  std::cout << "float type" << std::endl;
}

template <typename T>
typename std::enable_if<!std::is_floating_point_v<T>>::type k(T const& x) {
  std::cout << "not float type" << std::endl;
}


int main() {
  f(10);
  f(10.0);

  h(10);
  h(10.0);

  k(10);
  k(10.0);
}
//output:
//not float type
//float type
//not float type
//float type
//not float type
//float type

使用 constexpr if 实现 SFINAE

当然，可以利用 c++17 的 if constexpr特性， 让代码更加简洁和直观：

#include <iostream>
#include <type_traits>

class MyClass {
public:
  template <typename T>
  static void f(T t) {
    if constexpr (std::is_floating_point<T>::value) {
      std::cout << "call f(float)" << std::endl;
    } else {
      std::cout << "call f(integral)" << std::endl;
    }
  }
};

int main() {
  MyClass::f(10);
  MyClass::f(10u);
  MyClass::f(0.2);
}
